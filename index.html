How the OAuth flow works

Use OAuth 2.0 for Gmail — no mailbox password is ever collected. The frontend redirects to the backend, which exchanges the authorization code for tokens and fetches mail via the Gmail API.

Google / Gmail

  1. Create OAuth credentials in Google Cloud Console.
  2. Add redirect URI: https://your-backend-name.vercel.app/api/auth/google/callback.
  3. Request scopes: openid email profile gmail.readonly.
  4. Use the Gmail API to list, read, and label emails.

Backend on Render (free)

  1. Push backend/ folder to GitHub.
  2. Create a Web Service on Render, set env vars.
  3. Update CONFIG.BACKEND_URL in this file.
  4. Reload — auth is live.
